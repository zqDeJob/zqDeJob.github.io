动手写代码前，先把基础环境搭稳。下面是一份个人向的「最小工具链」清单：版本管理、运行时、镜像源——装对一次，后面会省很多折腾。

## 必备两件套

| 工具                                  | 作用                   |
| ----------------------------------- | -------------------- |
| [Git](https://git-scm.com/install/) | 代码版本管理、协作与部署         |
| Node.js                             | 前端/脚本生态的运行时（很多项目依赖它） |

**Git** 安装很直接：到官网下载对应系统安装包，一路下一步即可。

**Node** 则要多留点心：版本多、项目之间常要切换；官方安装包又常走国外源，国内网络下容易下载失败或极慢。因此通常会再配 **nvm**（管 Node 版本）和 **nrm**（管 npm 源）。

## Git：基本配置与 SSH 密钥

装好 Git 后，先设置提交时显示的身份（邮箱建议与 GitHub 账号一致）：

```bash
git config --global user.name "你的名字"
git config --global user.email "your_email@example.com"
```

### 生成 SSH 密钥

用 SSH 地址（`git@github.com:...`）拉代码、推送时，需要本机一对密钥：**私钥留在电脑**，**公钥填到 GitHub**。

在终端执行（把邮箱换成你的）：

```bash
ssh-keygen -t rsa -b 4096 -C "your_email@example.com"
```

提示保存路径时直接回车即可，默认生成在 `~/.ssh/`（Windows 一般为 `C:\Users\你的用户名\.ssh\`）。是否设置 passphrase（密码短语）可选：有则更安全，每次使用密钥可能要输入；图省事可留空。

会生成两个文件：

| 文件 | 说明 |
|------|------|
| `id_rsa` | 私钥，**不要**发给任何人、不要提交到仓库 |
| `id_rsa.pub` | 公钥，复制到 GitHub / GitLab 等平台 |

### 查看公钥

**Linux / macOS / Git Bash：**

```bash
cat ~/.ssh/id_rsa.pub
```

**Windows PowerShell：**

```powershell
Get-Content $env:USERPROFILE\.ssh\id_rsa.pub
```

复制终端输出的**整行**（以 `ssh-rsa` 开头，以邮箱结尾），打开 GitHub → **Settings → SSH and GPG keys → New SSH key**，标题随意，Key 里粘贴保存。

### 验证是否生效

```bash
ssh -T git@github.com
```

首次连接输入 `yes`，成功时会提示已认证（例如 `Hi username! You've successfully authenticated...`）。之后 `git clone git@github.com:xxx/xxx.git`、`git push` 就不必每次输 HTTPS 密码了。

## nvm：切换 Node 版本

文档与下载入口：[nvm 中文站 - 下载](https://nvm.uihtm.com/doc/download-nvm.html)

### Linux / macOS

网络正常时可用官方脚本；不畅时可用国内镜像脚本：

```bash
# 官方
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.3/install.sh | bash

# 国内替代
curl -o- https://gitee.com/RubyMetric/nvm-cn/raw/main/install.sh | bash

chmod +x ~/.nvm/nvm.sh
source ~/.bashrc
```

安装后可用 `nvm install --lts`、`nvm use <版本>` 在项目间切换 Node。

### Windows

Windows 上若执行 `nvm install lts` 长期失败，多半是下载源问题。到 nvm 安装目录找到 `settings.txt`（示例路径：`F:\nvm\nvm\settings.txt`），可改为类似配置：

```ini
root: F:\nvm\nvm
path: F:\nodejs
node_mirror: https://npmmirror.com/mirrors/node/
npm_mirror: https://npmmirror.com/mirrors/npm/
```

保存后重新执行 `nvm install lts` 再试。

#### 环境变量（`nvm` 命令找不到时）

在 **系统环境变量** 中确认：

| 变量            | 建议值（按你的实际安装路径修改）                  |
| ------------- | --------------------------------- |
| `NVM_HOME`    | `F:\nvm\nvm`                      |
| `NVM_SYMLINK` | `F:\nodejs`                       |
| `Path`        | 包含 `%NVM_HOME%` 和 `%NVM_SYMLINK%` |

改完环境变量后**重新打开终端**（必要时注销或重启），再执行 `nvm version` 验证。

#### PowerShell 提示「禁止运行脚本」（npm 无法使用）

用 nvm 装好 Node 后，在 **PowerShell** 里执行 `npm -v` 或 `npm i` 时，可能报错：

```text
npm : 无法加载文件 F:\nodejs\npm.ps1，因为在此系统上禁止运行脚本。
```

原因是 Windows 默认的 **执行策略** 较严，不允许运行 `npm.ps1` 这类脚本。Node 已装好，只是当前终端拦住了 npm。

在 **PowerShell**（建议用管理员或普通窗口均可）执行：

```powershell
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
```

出现确认提示时输入 **`Y`** 回车。`CurrentUser` 只影响你的账号，不必改整机策略；`RemoteSigned` 允许本地脚本运行，从网络下载的脚本需签名，日常开发够用。

然后**关掉终端再开**，验证：

```powershell
npm -v
```

若仍报错，可改用 **命令提示符（cmd）** 或 **Git Bash** 使用 npm，它们不依赖 `.ps1` 脚本；长期用 PowerShell 的话，还是建议按上面改执行策略。

## nrm：切换 npm 镜像源

装好 Node 与 npm 后，用 **nrm** 在多个 registry 之间切换，避免默认源过慢：

```bash
npm i -g nrm
```

常用命令：

```bash
nrm ls              # 列出可用源
nrm use taobao      # 切换到淘宝镜像（名称以 ls 列表为准）
```

国内开发建议先 `nrm ls`，选延迟低的源再 `use`；若某源失效，换另一个即可。

***

## 建议的安装顺序

1. 安装 **Git**，配置用户名与邮箱，生成 SSH 公钥并添加到 GitHub
2. 安装 **nvm**，用其安装一个 LTS 版 Node
3. 全局安装 **nrm**，切换 npm 镜像
4. 进入具体项目后，再按仓库说明安装依赖（如 `npm install` 或 `pnpm install`）

***

## 小结

- **Git**：版本管理的底座；记得 `user.name` / `user.email`，以及 SSH 密钥（`ssh-keygen` + 公钥加到 GitHub）。
- **nvm**：解决「多项目、多 Node 版本」问题；Windows 注意镜像、`settings.txt`、环境变量，以及 PowerShell 下 npm 的执行策略。
- **nrm**：解决「npm 装包慢、超时」问题，一条命令换源。

环境搭好后，克隆仓库、装依赖、跑脚本会顺畅很多。若你也有踩坑记录，欢迎在评论区补充。
